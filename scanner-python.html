Le problème à résoudre

Les scanners de vulnérabilités classiques comme Nikto ou OWASP ZAP analysent principalement le HTML statique. Or, les applications web modernes reposent sur JavaScript, des requêtes AJAX et des frameworks comme React ou Vue. Ces applications sont invisibles pour les outils traditionnels.

L'objectif était de créer un outil capable d'interagir avec une page web comme un vrai navigateur — en cliquant sur des boutons, en remplissant des formulaires — pour détecter des vulnérabilités qui ne s'activent qu'à l'exécution.

Note éthique : Cet outil est développé à des fins éducatives et ne doit être utilisé que sur des environnements de test ou avec une autorisation écrite explicite. Toute utilisation non autorisée est illégale.

Architecture & fonctionnement

L'outil s'articule autour de Playwright, la librairie de Microsoft pour contrôler un navigateur headless (Chromium, Firefox) via Python. Il simule un utilisateur réel et peut interagir avec des éléments JavaScript dynamiques.

# Initialisation du scanner avec Playwright
from playwright.sync_api import sync_playwright
import json, re

class VulnScanner:
    def __init__(self, target_url):
        self.target = target_url
        self.findings = []
        self.playwright = sync_playwright().start()
        self.browser = self.playwright.chromium.launch(headless=True)

    def scan_sqli(self, page, input_selector):
        # Payloads SQL Injection classiques
        payloads = [
            "' OR '1'='1",
            "'; DROP TABLE users;--",
            "' UNION SELECT 1,2,3--"
        ]
        for payload in payloads:
            page.fill(input_selector, payload)
            page.keyboard.press("Enter")
            # Analyse de la réponse

Le module XSS teste l'injection de scripts dans tous les champs de formulaire détectés, et vérifie si le script est bien exécuté par le navigateur (et non simplement affiché comme texte). Cette différence est cruciale et souvent ratée par les scanners statiques.

def scan_xss(self, page):
    xss_payloads = [
        '<script>alert("XSS")</script>',
        '"><img src=x onerror=alert(1)>',
        "javascript:alert('XSS')"
    ]
    # Détection via dialog JavaScript
    page.on("dialog", lambda d: (
        self.findings.append({
            "type": "XSS",
            "severity": "HIGH",
            "payload": d.message
        }), d.dismiss()
    ))

Vulnérabilités détectées

  • SQL Injection (SQLi) — détection par analyse des messages d'erreur SQL dans la réponse HTTP et comportements anormaux.
  • Cross-Site Scripting (XSS) — détection via les événements dialog du navigateur et modification du DOM.
  • CSRF (Cross-Site Request Forgery) — vérification de la présence de tokens CSRF dans les formulaires et headers.
  • Informations sensibles exposées — scan des headers HTTP, commentaires HTML et messages d'erreur verbeux.

Résultats & rapport

Le scanner génère un rapport JSON structuré avec chaque vulnérabilité, sa sévérité (CRITICAL / HIGH / MEDIUM / LOW), le payload utilisé et des recommandations de correction. Un exemple de sortie terminale ci-dessous :

Ce que ce projet m'a appris

Au-delà du code, ce projet m'a permis de comprendre concrètement pourquoi ces vulnérabilités existent, comment les développeurs les introduisent par inadvertance, et comment les corriger. C'est cette double compréhension — attaquant/défenseur — qui est au cœur de la cybersécurité.

Compétence clé développée : penser comme un attaquant pour mieux défendre. La maîtrise des vecteurs d'injection est essentielle pour un auditeur de sécurité.
