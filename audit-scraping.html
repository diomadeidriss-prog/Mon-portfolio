Le problème de la surface d'attaque

Avant de lancer une cyberattaque, tout attaquant commence par une phase de reconnaissance (recon). Il collecte des informations sur la cible : adresses email, sous-domaines, technologies utilisées, fichiers exposés, identifiants divulgués sur des forums.

L'objectif de ce projet était de simuler cette phase de recon en étant du côté défenseur — pour trouver et corriger les expositions avant qu'un acteur malveillant ne les exploite.

Cadre éthique : Ce projet a été réalisé dans un cadre strictement éducatif, sur des cibles fictives ou avec une autorisation explicite. L'OSINT offensif sans consentement est illégal (art. 323-1 Code pénal français).

Ce que l'outil recherche

L'outil combine plusieurs sources d'information pour dresser un portrait complet de la surface d'attaque d'une organisation :

  • Emails d'employés exposés sur des pages Contact, LinkedIn, forums techniques
  • Sous-domaines découverts via DNS brute-force et certificats TLS (crt.sh)
  • Technologies détectées (CMS, frameworks, versions) via headers HTTP et metadata
  • Fichiers sensibles exposés (.env, backup.zip, .git/, config.php...)
  • Credentials leakés référencés sur des bases comme HaveIBeenPwned
  • Appareils exposés sur internet via Shodan (caméras, routeurs, serveurs)

Implémentation technique

L'outil est structuré en modules indépendants, chacun responsable d'un vecteur de collecte, orchestrés par un moteur central qui agrège et déduplique les résultats.

import re, requests
from bs4 import BeautifulSoup

def harvest_emails(domain, depth=2):
    """Collecte les emails exposés sur le domaine cible"""
    found_emails = set()
    email_pattern = re.compile(
        r'[a-zA-Z0-9._%+-]+@[a-zA-Z0-9.-]+\.[a-z]{2,}'
    )
    urls_to_visit = [f"https://{domain}"]
    visited = set()

    for url in urls_to_visit[:depth*10]:
        if url in visited: continue
        response = requests.get(url, timeout=5)
        soup = BeautifulSoup(response.text, 'html.parser')
        
        # Scan dans le texte brut
        emails = email_pattern.findall(response.text)
        found_emails.update(emails)
        
        # Scan des balises mailto:
        for link in soup.find_all('a', href=True):
            if 'mailto:' in link['href']:
                found_emails.add(link['href'].replace('mailto:', ''))
        visited.add(url)

    return list(found_emails)
# Détection de fichiers sensibles exposés
SENSITIVE_PATHS = [
    ".env", ".git/config", "backup.sql", "backup.zip",
    "config.php.bak", "phpinfo.php", "wp-config.php.bak",
    "admin/config.yml", ".DS_Store", "web.config"
]

def check_exposed_files(base_url):
    exposed = []
    for path in SENSITIVE_PATHS:
        url = f"{base_url}/{path}"
        r = requests.head(url, timeout=3, allow_redirects=False)
        if r.status_code == 200:
            exposed.append({
                "url": url, "size": r.headers.get("Content-Length"),
                "risk": "CRITICAL"
            })
    return exposed

Rapport d'audit généré

À la fin du scan, l'outil génère un rapport structuré (JSON + PDF) avec :

  • Résumé exécutif avec score de risque global
  • Détail de chaque finding avec niveau de criticité
  • Recommandations concrètes de remédiation
  • Timeline de découverte des informations

Résultat sur cible de test : 23 emails exposés, 4 sous-domaines fantômes non maintenus, 2 fichiers .env accessibles publiquement — tous corrigés après rapport.
